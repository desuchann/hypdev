What motivates me?

Resolving issues for good!

From a DevOps perspective, this means getting to the bottom of complex issues by debugging the issue, exactly pinpointing the root cause, and providing suggestions on how to prevent it going forward plus introducing telemetry where possible in the hope of henceforth detecting it before it becomes an issue.

To build optimal code, Software Engineers should always write code with an SRE viewpoint i.e. building extremely supportable code with clear logging and error handling without compromising on readability and end user experience.
